What happens when you initialize a repository? Why do you need to do it?

Git creates metadata files in hidden folder .git. To be able to star versioning the files

How is the staging area different from the working directory and the repository? What value do you think it offers?

It keeps track of the files added using git add command while the working directory just contains all the files in the repository, wether they are tracked or untracked

How can you use the staging area to make sure you have one commit per logical change?

By comparing the changes in the staging area with the current commit

What are some situations when branches would be helpful in keeping your history organized? How would branches help?

When you want to test some changes and do not want to modify the master branch. They allow to create other repositories to which make commits without affecting the original one
